# draftpilot — 競合分析

## 直接競合

| ツール | 強み | 弱み | draftpilot 優位性 |
|--------|------|------|-------------------|
| Hypefury | スレッド/キュー機能充実 | 英語中心、タイムライン表示あり | JP対応 + タイムライン非表示 |
| Buffer | チーム向け、多SNS | AI 下書き弱い、UIが重い | AI生成が中心設計 |
| TweetHunter | AI生成あり、分析強 | $49/mo~、JP UI なし | ¥980 = 1/5コスト |
| Typefully | クリーンUI、スレッド | X 連携のみ、MCP なし | MCP ファースト |

## 間接競合

- ChatGPT / Claude直接: コピペ手動 → UX摩擦大
- X 公式 Premium: AI 機能なし
- Notion AI: 投稿に特化していない

## 差別化マトリクス

```
                   AI 生成
                   強い↑
                     │ draftpilot ★
           MCP ◄────●────► タイムライン表示
                     │
                   弱い↓
```

## 採用不採用の分岐点

採用: 「AIに下書きを任せたい」「タイムラインを見たくない」「Claude Codeから投稿したい」
不採用: 「複数SNS同時運用したい」「チームで承認フロー必要」「分析ダッシュボードが要件」

## Moat (競争優位の堀)

1. **MCP 統合**: 現状唯一の X × MCP サーバー提供 (2026年5月時点)
2. **タイムライン不表示**: UX 設計として不可逆な制約 → 模倣困難
3. **JP クリエイター特化**: 大手は EN 優先、JP 対応は後付け
4. **Founding 100 コミュニティ**: 初期採用者との関係性
